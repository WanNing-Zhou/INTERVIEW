CSS Postion(定位)

position属性制定了元素的定位类:
position属性的五个值:
  • static
  • relative
  • fixed
  • absolute
  • sticky
    • 元素可以使用的顶部,底部,左侧和右侧属性的定位,然而,这些属性无法工作,除非是先设定的postion属性,他们也有不同的工作方式,这取决于定位方法

static定位

HTML元素默认值,即没有定位,遵循正常的文档流对象
静态定位的元素不会受到top,button,left,right影响
这是一个static定位元素

fixed定位

元素的位置相对于浏览器窗口是固定位置
即使是窗口滚动的它也不会移动
Fixed定位在IE7和IE8下需要描述!DOCTYPE才能支持
Fixed定位使元素的位置与文档六无关,因此不占据空间
Fixed定位的元素和其他元素重叠

这是开启fixed定位的元素

注意: IE7 和 IE8 支持只有一个 !DOCTYPE 指定固定值.

relative定位

相对定位元素的定位是相对其正常位置

相对定位会按照元素的原始位置对该元素进行移动。

样式 "left:-20px" 从元素的原始左侧位置减去 20 像素。

样式 "left:20px" 向元素的原始左侧位置增加 20 像素。

absolute定位

绝对定位的元素的位置相对于最近的已定位父元素,如果没有已定位的父元素,那么他的位置相对于 html
注意:
absolute定位使元素的位置与文档流无关,因此不占据空间

sticky定位

sticky 英文自卖你意思是粘,粘贴, 所以可以把它称之为粘性定位
postion: sticky; 基于用户的滚动位置来定位
粘性定位的元素是依赖于用户的滚动,在 position:relative 与 position:fixed定位之间切换
它的行为就像position:relative;而当页面滚动超出目标区域时, 它的表现就像position:fixed; 它会固定在目标位置
元素定位表现在跨越特定阈值为相对定位,之后为固定定位
这个特定域指指的是top,right,bottom或left之一,换言值,指定top,right,bottom或left四个阈值其中之一,才可使粘性定位生效,否则其行为与相对定位相同
注意:internet Explorer,Edge 15以及更早IE版本不支持sticky,Safari需要使用 -webkit- prefix
